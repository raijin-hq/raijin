# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Raijin, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please email: **security@raijin.dev**

Include:
- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will acknowledge your report within 48 hours and aim to provide a fix or mitigation within 7 days for critical issues.

## Scope

This policy covers:
- The Raijin terminal emulator (`raijin-app`)
- The Inazuma framework (`inazuma`)
- Shell integration hooks (`raijin-shell`)
- PTY handling and terminal emulation (`raijin-terminal`, `raijin-term`)

## Supported Versions

As Raijin is in active development and has not yet reached a stable release, security fixes are applied to the `main` branch.
